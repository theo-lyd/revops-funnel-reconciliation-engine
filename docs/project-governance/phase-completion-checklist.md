# Phase Completion Checklist And Stop-Gate

Use this checklist at the end of every phase before requesting go/no-go confirmation.

## Completion checklist
- Scope for current phase is implemented.
- Tests and quality checks for the phase pass.
- Required report file is created under docs/reports/phase-X.
- Issues log is updated with any new incidents.
- Command logs are updated with all relevant commands.
- Commit is created with grouped message.
- Commit is pushed to origin.

## Stop-gate action
After checklist completion:
- Summarize outcomes, open risks, and evidence links.
- Ask for explicit confirmation to proceed to the next phase.
